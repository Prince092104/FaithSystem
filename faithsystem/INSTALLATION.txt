================================================================================
DATABASE SCHEMA
================================================================================

MEMBERS TABLE
  - id (INT) - Primary key
  - first_name, last_name (VARCHAR)
  - email (VARCHAR) - Unique
  - phone (VARCHAR)
  - date_of_birth (DATE)
  - join_date (DATE)
  - address, city, state, zipcode (VARCHAR)
  - status (ENUM: Active, Inactive, Suspended, Expired)
  - membership_type (ENUM: Regular, Premium, Student, Senior)
  - membership_fee (DECIMAL)
  - date_created, date_updated (TIMESTAMP)

MEMBER_FILES TABLE
  - id (INT) - Primary key
  - member_id (INT) - Foreign key
  - file_name, file_path (VARCHAR)
  - file_type, file_size (VARCHAR/INT)
  - upload_date (TIMESTAMP)

STATUS_HISTORY TABLE
  - id (INT) - Primary key
  - member_id (INT) - Foreign key
  - old_status, new_status (VARCHAR)
  - change_reason (VARCHAR)
  - changed_date (TIMESTAMP)

MEMBERSHIP_TYPES TABLE
  - id (INT) - Primary key
  - type_name (VARCHAR)
  - description (VARCHAR)
  - annual_fee (DECIMAL)
  - benefits (TEXT)

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Database connection error
Solution: 
  - Check MySQL is running in XAMPP
  - Verify credentials in config/config.php
  - Run setup_verify.php to check configuration

Problem: Cannot see styles or layout
Solution:
  - Clear browser cache (Ctrl+F5)
  - Verify css/style.css file exists
  - Check browser console for errors

Problem: Members page shows error
Solution:
  - Ensure database tables are created
  - Run database.sql in phpMyAdmin
  - Check database connection settings

Problem: Reports not exporting
Solution:
  - Verify write permissions in project directory
  - Check browser pop-up settings
  - Try different export format
